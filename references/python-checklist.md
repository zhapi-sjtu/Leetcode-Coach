# Python 检查清单

在严格复盘 Python 解法时使用这些问题。

## 数据结构

- 为什么这里使用 `dict`、`set`、`list`、`deque`、`heapq` 或 `defaultdict`？
- 查找、插入、删除的平均时间复杂度分别是多少？
- `list.pop(0)` 会不会有问题？是否应该使用 `deque.popleft()`？
- `heapq` 实现的是小根堆还是大根堆？

## 可变对象

- 代码是否修改了输入？
- 是否存在别名问题，例如 `grid2 = grid` 或 `[[0] * n] * m`？
- 递归中是否错误共享了可变的路径列表？

## 循环和边界

- 这个循环为什么从这些下标开始和结束？
- `range` 的边界是包含还是不包含？
- `i + 1`、`mid + 1` 或 `right - 1` 会不会越界？

## 排序和 key

- `key=` 函数按什么排序？
- 排序稳定性在这里是否重要？
- 排序开销是否可以接受？

## 递归

- 递归终止条件是什么？
- 递归深度会不会超过 Python 默认限制？
- 是否应该改成迭代版 DFS/BFS？

## 动态规划和初始化

- DP 表初始化是否正确？
- 每一行是否相互独立？
- 是否可以使用滚动数组优化？

## 常见写法

- 解释 `enumerate`、`zip`、切片、列表推导式、`collections.Counter`、`defaultdict(list)` 的作用。
- 如果使用 `float("inf")`，还能用什么值替代？
- 如果使用整数除法，为什么用 `//` 而不是 `/`？

## 代码质量

- 变量名是否能体现算法含义？
- 能否在不隐藏逻辑的前提下简化代码？
- 哪一行最可能产生 bug？
