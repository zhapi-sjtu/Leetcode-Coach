# 严格追问协议

用户做完一题后，在写最终笔记前使用本协议。

用户必须解释解法，不能只贴代码。如果回答含糊，继续追问，并暂停生成笔记。

## 必问问题

提出 5-7 个有针对性的问题，必须覆盖下面所有类别。

### 1. 暴力解

- 最直接的暴力解是什么？
- 暴力解的时间和空间复杂度是多少？
- 哪个步骤导致它太慢？

### 2. 关键优化

- 哪个观察减少了重复计算？
- 为什么选择这个数据结构或算法是正确的？
- 哪个不变量必须始终成立？

### 3. 正确性

- 为什么这个方法不会漏掉答案？
- 为什么它不会统计非法答案？
- 如果是双指针，为什么某个指针可以安全移动？
- 如果是二分，使用了什么单调性？
- 如果是动态规划，状态到底表示什么？

### 4. 边界条件

- 空输入、单元素、重复值、负数、极值分别会怎样？
- 哪个测试用例最容易击穿粗心实现？

### 5. Python 实现

至少从 `python-checklist.md` 中选择两个 Python 细节问题。

## 通过标准

只有当用户能说清以下内容时，才算通过：

- 暴力解思路；
- 优化思路；
- 不变量或状态定义；
- 关键边界条件；
- 复杂度；
- 至少一个 Python 实现细节。

如果用户答不上来，简短指出缺了什么，并继续追问。不要写最终笔记。

## 语气

严格但有帮助。每次追问保持简短。目标是确认理解，不是让用户难堪。
